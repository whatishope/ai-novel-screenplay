# 项目规划

本文档用于记录 AI Novel Screenplay 的 MVP 交付计划、已完成范围和后续迭代方向。

## 1. 项目目标

AI Novel Screenplay 的目标是提供一条从小说文本到结构化剧本 YAML 的可演示工作流。MVP 阶段优先保证流程完整、结果可编辑、结构可校验、文件可导出。

核心目标：

- 支持用户上传或粘贴小说文本。
- 自动拆分章节，提取角色，规划场景。
- 生成包含 `metadata`、`characters`、`relationships`、`scenes`、`production` 的 YAML 剧本。
- 提供前端工作台完成生成、编辑、校验、复制和导出。
- 保持本地演示稳定，即使没有配置真实 LLM Key 也能使用 mock 数据运行。

## 2. MVP 交付范围

MVP 交付内容分为文档、后端、前端、验证四部分。

### 2.1 文档

- 产品设计文档：说明目标用户、核心痛点、功能范围和技术路线。
- YAML Schema 文档：定义剧本 YAML 的顶层结构、字段含义和校验规则。
- API 文档：说明后端接口、请求响应格式和本地启动方式。
- Demo 指南：说明演示前置条件、操作步骤、验收清单和常见问题。
- README：提供项目简介、技术栈、本地启动、测试构建和文档入口。

### 2.2 后端

- Spring Boot 3 后端项目初始化。
- 小说 `.txt` 上传和 UTF-8 文本读取。
- 小说文本章节拆分。
- 角色提取服务，支持 mock LLM 响应。
- 场景规划服务，支持 mock LLM 响应。
- YAML 生成服务，包含角色关系、动作、对白、来源追踪和制作信息。
- YAML 校验服务，返回错误、警告和统计数量。
- 从原始文本到 YAML 的一键生成接口。
- 全局异常处理、参数校验、CORS 配置和健康检查。

### 2.3 前端

- Vue 3 + Vite 工作台初始化。
- 小说标题、文本粘贴、示例载入和 `.txt` 上传。
- 一键生成完整剧本结构。
- 章节、角色、场景结构化展示。
- YAML 在线编辑、复制、校验、导出和重置。
- 校验结果展示，包含错误、警告、角色数量、关系数量和场景数量。
- 本地开发代理，将 `/api` 请求转发到后端。

### 2.4 验证

- 后端单元测试和控制器测试。
- 前端生产构建检查。
- 本地接口冒烟测试。
- 本地演示流程检查。

## 3. 已完成状态

截至当前版本，MVP 主流程已经完成：

1. 用户输入小说文本。
2. 后端拆分章节。
3. 后端提取角色。
4. 后端规划场景。
5. 后端生成 YAML。
6. 前端展示结构化结果。
7. 用户编辑并校验 YAML。
8. 用户复制或导出 `screenplay.yaml`。

当前验证命令：

```bash
cd backend
mvn clean test
```

```bash
cd frontend
npm.cmd run build
```

## 4. 迭代记录

建议继续保持小 PR 节奏，每个 PR 聚焦一个明确主题：

- `docs/*`：文档、README、示例或演示指南。
- `feat/*`：新增可见功能。
- `fix/*`：修复缺陷或兼容性问题。
- `test/*`：补充测试。
- `chore/*`：构建、配置或依赖整理。

每个 PR 建议包含：

- 功能描述。
- 实现思路。
- 测试方式。
- 是否影响前端、后端或文档。

## 5. 后续优化方向

MVP 完成后，可以从以下方向继续扩展：

- 接入真实 LLM 提供商，并增加 Key 配置说明。
- 增加生成进度状态，避免长文本生成时页面等待不明确。
- 支持单个场景重新生成和局部改写。
- 增强角色一致性、剧情遗漏和来源追踪校验。
- 增加 YAML Schema 自动化校验文件。
- 支持导出更多格式，例如 Markdown、JSON 或分镜表。
- 增加项目保存、历史版本和多项目管理。
- 提供 Docker Compose 本地演示环境。

## 6. 当前不做范围

以下能力暂不属于 MVP：

- 用户登录和权限管理。
- 数据库持久化。
- 多人协作。
- 付费、配额和审计。
- 复杂任务队列。
- 生产环境部署。

这些能力可以在 MVP 演示稳定后再按实际需求拆分为独立迭代。
